# `_meta.json`

A `_meta.json` next to your MDX files lets you override the
auto-derived sidebar labels and ordering for sibling files and
folders.

## Schema

```ts
type MetaJson = {
  [segment: string]:
    | string                                            // shorthand: { "foo": "Custom label" }
    | ({ label?: string; order?: number; hidden?: boolean } & {
        // Any additional keys are passed through to the theme as
        // `NavNode.meta` — bundoc does not interpret them.
        [extra: string]: unknown;
      });
};
```

The keys are *segments*: leaf filenames without `.mdx`, or folder
names. The values are either a plain string label or an object with
explicit fields.

Any keys beyond `label`/`order`/`hidden` are passed through to the
theme verbatim via `NavNode.meta`. Use this for icons, badges,
separator hints, or any other per-entry data your theme needs.

## Examples

### Reorder + relabel

```json title="content/guides/_meta.json"
{
  "writing-content": { "label": "Writing content", "order": 0 },
  "i18n": { "label": "Internationalisation", "order": 1 },
  "deploy": { "label": "Deploy", "order": 99 }
}
```

### String shorthand

When you only want to relabel:

```json
{
  "i18n": "Internationalisation"
}
```

### Pass-through theme metadata

```json title="content/_meta.json"
{
  "core": {
    "label": "Core",
    "order": 0,
    "icon": "cpu",
    "badge": "new"
  }
}
```

These extras land on the corresponding `NavNode.meta`:

```tsx
{nav.tree.children.map((node) => (
  <SidebarItem
    key={node.seg}
    label={node.label}
    icon={node.meta?.icon as string | undefined}
    badge={node.meta?.badge as string | undefined}
  />
))}
```

### Hidden entries

```json
{
  "drafts": { "hidden": true }
}
```

A hidden entry stays accessible by URL but doesn't appear in the
sidebar or in `useNav().flat` (and so doesn't show up in prev/next).

<Callout type="tip">
  Prefer renaming the file to `_drafts.mdx` if you want it ignored
  entirely. `hidden: true` keeps the route reachable for direct
  linking.
</Callout>

## Precedence

Sidebar order is resolved in this priority:

1. `_meta.json` `order` for the segment.
2. `frontmatter.sidebar.order` on the file itself.
3. Fallback `0`.

Labels resolve in this priority:

1. `_meta.json` `label` for the segment.
2. `frontmatter.sidebar.label`.
3. `frontmatter.title`.
4. Auto-derived from the segment name (`getting-started` → *Getting Started*).
