# Search

bundoc builds one Orama index per locale at build time and serves it
from `dist/_bundoc/search/<locale>.json`. The `useSearchIndex()` hook
lazy-fetches and queries it client-side ‚Äî no server, no API key.

## What gets indexed

Every (route, locale) pair in the manifest, with one exception:
**synthesised locale fallbacks are skipped**. If a page only exists in
the default locale, only the default-locale index includes it ‚Äî the
runtime falls through to the default index on miss.

Beyond that, the inclusion policy is yours via `search.filter` in
`bundoc.config.ts`.

## The `search.filter` predicate

```ts
import { defineConfig, type SearchablePage } from "@clusterize/bundoc/config";

export default defineConfig({
  // ‚Ä¶
  search: {
    filter: (page: SearchablePage) => page.frontmatter.search !== false,
  },
});
```

`SearchablePage` has:

| Field | Type | Notes |
|---|---|---|
| `route` | `string` | Route path with leading slash (`"/getting-started/installation"`). |
| `locale` | `string` | BCP-47-style code (`"en"`, `"de"`). |
| `title` | `string` | The page's display title (frontmatter `title` or derived). |
| `frontmatter` | `Record<string, unknown>` | The page's raw frontmatter. |
| `fallback` | `boolean` | Always `false` for predicate calls ‚Äî fallback rows are filtered out before the predicate runs. The field is present for API symmetry. |

Return `true` to include the page in the index, `false` to skip it.

### Default predicate

If you omit `search.filter`, bundoc applies:

```ts
(page) => page.frontmatter.search !== false
```

This is the conventional opt-out ‚Äî any page with `search: false` in
its frontmatter is excluded.

You can import and compose with it:

```ts
import { defaultSearchFilter } from "@clusterize/bundoc/config";

export default defineConfig({
  // ‚Ä¶
  search: {
    filter: (p) =>
      defaultSearchFilter(p) &&
      !p.route.startsWith("/internal/") &&
      p.frontmatter.draft !== true,
  },
});
```

## Indexed frontmatter fields

By default, each page's `description` frontmatter is folded into the
indexed body text ‚Äî so a page is findable by phrases that only appear
in its description, not in its prose. The folded text is concatenated
into the existing `text` field, so query-time boosts on `title` and
`heading` are unchanged.

Extend the list to fold extra keys, or pass `[]` to disable:

```ts
import { defineConfig } from "@clusterize/bundoc/config";

export default defineConfig({
  // ‚Ä¶
  search: {
    frontmatterFields: ["description", "keywords"],
  },
});
```

Supported value shapes:

- `string` ‚Äî included verbatim.
- `string[]` ‚Äî each element is included.
- Anything else (numbers, objects, booleans) is ignored.

Pass `frontmatterFields: []` to opt out entirely and restore the
pre-0.3 behaviour where only the body text is indexed.

## Examples

### Exclude a route prefix

```ts
search: {
  filter: (p) => !p.route.startsWith("/api/"),
}
```

### Exclude drafts

```ts
search: {
  filter: (p) => p.frontmatter.draft !== true,
}
```

### Locale-aware

```ts
search: {
  filter: (p) => p.locale === "en" || p.frontmatter.translated === true,
}
```
